## 왜 만들었는가

ChatGPT, Claude, Cursor 등 AI 코딩 도구가 대중화되면서
**코드를 직접 작성하지 않고도 프로젝트를 만드는 "바이브코딩" 시대**가 열렸다.

문제는, AI가 생성한 코드에는 종종 다음이 포함된다:
- 하드코딩된 API 키, 비밀번호
- `eval()`, `innerHTML` 같은 위험 패턴
- `.env` 파일이 `.gitignore`에서 빠진 설정 실수

기존 보안 도구(Snyk, Semgrep, Gitleaks 등)는 **개발 경험이 있는 사람을 위해 설계**되어 있어,
초보자가 결과를 보고 "그래서 뭘 어떻게 고치라는 건지" 알기 어렵다.

**VibeScan은 이 간극을 메운다:**
- 모든 이슈에 "왜 위험한지(why)"와 "어떻게 고치는지(fix)"를 포함
- 100% 로컬 실행 — 코드가 외부로 전송되지 않음
- 한국어/영어 설명형 리포트 제공

## 아키텍처

```
사용자 입력 (vibescan scan .)
        │
        ▼
  ┌─────────────┐
  │  CLI Parser  │  Typer 기반 명령어 파싱
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │  Collector   │  파일 수집, .gitignore 파싱, 인코딩 처리
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Rule Engine  │  20개 규칙 파일, 플러그인 구조
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │ Aggregator   │  심각도 필터링, 그룹핑, exit code 결정
  └──────┬──────┘
         ▼
  ┌─────────────┐
  │  Reporter    │  Console / JSON / HTML 출력
  └─────────────┘
```

### 설계 포인트

- **플러그인 기반 Rule Engine**: 새 규칙 추가 시 클래스 하나만 작성하면 자동 등록. 기존 코드 수정 불필요 (OCP 준수)
- **ProjectContext 공유 객체**: 파일 목록, .gitignore 결과를 한 번만 수집하고 모든 규칙이 공유
- **Issue 객체에 why/fix 내장**: 단순 경고가 아닌 교육적 메시지 — 이 프로젝트의 핵심 차별점
- **CI/CD 호환**: CRITICAL/HIGH 발견 시 exit code 1 반환 → GitHub Actions 연동 가능

## 핵심 기능 상세

### 1. Secret 탐지 (14개 카테고리)

환경변수 파일, 클라우드 인증, Docker/CI 설정, SSH 키, 프론트엔드 환경변수, IDE 설정,
모바일 앱 파일, 데이터베이스 파일, 문서 내 실수까지 — 실무에서 발생하는 거의 모든 유출 경로를 커버.

**오탐 방지 필터**: placeholder(`your-api-key-here`), 한국어 에러 메시지, 테스트 코드 등을 자동 제외.

### 2. 위험 코드 패턴 분석

Python/JavaScript/TypeScript의 위험 패턴 탐지:
- `eval()`, `exec()`, `subprocess(shell=True)`
- `innerHTML`, `dangerouslySetInnerHTML`
- SQL 인젝션 (f-string/template literal 쿼리)
- `jwt.decode()` without `jwt.verify()`

### 3. Git Hygiene 점검

`.gitignore`에 `.env`, `node_modules`, 키 파일 등이 누락되었는지 교차 검증.

### 4. 설명형 리포트

```
CRITICAL  config.py:23  하드코딩된 AWS 키 (AKIA...)

  Why: API 키가 코드에 직접 입력되어 있습니다.
       Git 커밋 시 누구나 키를 사용할 수 있습니다.
  Fix: 환경변수로 분리하세요.
       os.environ["AWS_ACCESS_KEY_ID"]를 사용하세요.
```

### 5. 다국어 지원 (i18n)

시스템 로케일 자동 감지. `--lang ko|en` 수동 지정 가능.
모든 이슈 설명/원인/해결책이 한국어와 영어로 제공.

## 어필 포인트 정리

### "문제 정의 → 해결"을 직접 설계했다
- 기존 보안 도구의 사용자 경험 문제를 분석하고
- "초보자를 위한 설명형 보안 리포트"라는 명확한 솔루션을 도출
- PRD, 아키텍처 문서를 직접 작성하고 구현까지 완수

### 외부 의존성 최소화
- 런타임 의존성 단 2개 (typer, rich)
- AI/ML 모델 없이 정규표현식과 패턴 매칭만으로 실용적 탐지 구현
- 설치 용량 최소화, 오프라인 동작 보장

### 확장 가능한 설계
- Rule Engine의 플러그인 구조 (OCP 준수)
- ProjectContext 패턴으로 데이터 수집과 분석 분리
- Reporter 인터페이스로 출력 형식 확장 가능

### 1인 풀스택 — CLI부터 웹사이트, IDE 확장까지
- Python CLI: 설계, 구현, 테스트
- 웹 랜딩 페이지: React 기반 디자인 및 구현
- VSCode Extension: TypeScript로 IDE 통합
- 인프라: 셀프호스팅 서버 + Caddy + 자동 HTTPS
- PyPI 패키지 배포

### 실사용 관점의 완성도
- 159개 테스트 케이스로 안정성 확보
- 오탐 방지 필터 (placeholder, 한국어 에러 메시지 등)
- CI/CD 연동 (exit code, JSON 출력)
- 한국어/영어 다국어 지원

## 면접 예상 질문과 답변

**Q: 왜 기존 도구(Snyk, Gitleaks)를 안 쓰고 직접 만들었나요?**
> 기존 도구는 경험 있는 개발자를 대상으로 합니다.
> VibeScan의 핵심은 "왜 위험한지"와 "어떻게 고치는지"를 초보자 눈높이에서 설명하는 것입니다.
> 도구 자체를 만든 게 아니라, 부족한 사용자 경험을 해결한 것입니다.

**Q: 정규표현식만으로 보안 스캐닝이 충분한가요?**
> AST 분석이나 taint analysis 대비 한계가 있지만,
> 실제 유출 사고의 대부분은 하드코딩된 키, .env 노출 같은 단순 패턴에서 발생합니다.
> 14개 카테고리의 실전 패턴과 오탐 필터를 조합해 실용적인 수준의 탐지를 달성했습니다.

**Q: 테스트는 어떻게 구성했나요?**
> 각 규칙별로 정상 탐지(true positive)와 오탐 방지(true negative) 케이스를 분리했습니다.
> fixture 기반으로 가짜 프로젝트 구조를 만들어 통합 테스트도 수행합니다.

**Q: 왜 100% 로컬인가요?**
> 보안 도구가 코드를 외부로 전송하면 그 자체가 보안 리스크입니다.
> import 목록에 네트워크 모듈이 없다는 것을 소스코드에서 직접 확인할 수 있습니다.
> 이 투명성이 사용자 신뢰의 근거입니다.
